Introduction

CI-Entity is CodeIgniter extension that help developers automatically create the interfaces of entities, tables or views, in their projects. The interfaces are common entity interfaces such as insertion, updating, deletion and searching. Just create table in database and a few line of code, entity recipes, CI-Entity will take care of the rest.

CI-Entity also create validations rules for each entity's attributes. For instance, a column which have decimal data type and not null will be construct validation rules as "required|decimal" automatically. However, If the additional validations is required, CI-Entity allow to add it in additionalValidation Class, this is described in additional validation rules section

The case study of using CI-Entity is demonstrated as Human Resources Development System. There are many tables involved to this system such as employees, classes, subjects, instructors, locations, training budgets, quizzes, etc. You do not need to write code for every single table (entity) for handle the operations such as updating, insertion, deletion, and form validation. This including interfaces for front-end users for every tables.


Pros and Cons of CI-Entity

Pros
  • Entity interface is easily to construct, just create table or view and a few lines of code.
  • Easily creating searching conditions input group, filter-row, for each entity.
  • Easily creating select2 component if a column is foreign key.
  • Datetime, date and time data type is automatically create datetime picker at front-end.
  • Automatically create validations rules for each attributes (table's column).
  • If a table structure have changed, column added or deleted, CI-Entity will be changed interface automatically.
  • Table and column name will not be disclosed to the public by using _POST and _GET method via AJAX.

Cons
  • Only supports MS SQL Server.
